# Backend Deployment Guide

Your Express.js backend can be deployed to several platforms. Choose based on your needs and budget.

---

## **QUICKSTART: Deploy to Railway (Easiest)**

### Step 1: Push Code to GitHub
```bash
cd c:\Users\haady\OneDrive\Desktop\ecommerce
git add .
git commit -m "Ready for deployment"
git push origin master
```

### Step 2: Create Railway Account
1. Go to https://railway.app
2. Click "Start a New Project"
3. Select "Deploy from GitHub Repo"
4. Authorize Railway with GitHub
5. Select your `ecommerce-site` repository

### Step 3: Configure Project
1. Click "Add Services" → Select "MySQL"
2. This creates a free MySQL database
3. Click "Deploy"

### Step 4: Set Environment Variables
1. Go to Project → Variables
2. Add these variables:
   ```
   PORT=3000
   DB_HOST=mysql (Railway provides this)
   DB_USER=root
   DB_PASSWORD=<auto-generated>
   DB_NAME=ecommerce_db
   JWT_SECRET=your-super-secret-key-change-this
   NODE_ENV=production
   ```
3. Railway shows all credentials - copy them
4. Click "Deploy"

### Step 5: Get Backend URL
- Railway provides: `https://your-project-xyz.railway.app`
- API endpoint: `https://your-project-xyz.railway.app/api`

## **Alternative 2: Deploy to Heroku**

### Step 1: Install Heroku CLI
```bash
# Download from https://devcenter.heroku.com/articles/heroku-cli
# Or on Windows:
choco install heroku-cli
```

### Step 2: Login to Heroku
```bash
heroku login
```

### Step 3: Create App
```bash
cd server
heroku create ecommerce-backend-api
```

### Step 4: Add MySQL Database
```bash
heroku addons:create cleardb:ignite
```
Gets: `CLEARDB_DATABASE_URL` environment variable

### Step 5: Set Environment Variables
```bash
heroku config:set JWT_SECRET=your-secret-key
heroku config:set NODE_ENV=production
heroku config:set DB_NAME=ecommerce_db
```

### Step 6: Deploy
```bash
git push heroku master
```

### Step 7: Get URL
```bash
heroku info
# Shows: https://ecommerce-backend-api.herokuapp.com
```

## **Alternative 3: Deploy to AWS (EC2)**

### Step 1: Launch EC2 Instance
1. Go to AWS console → EC2
2. Launch Ubuntu 22.04 instance (t2.micro for free tier)
3. Create security group:
   - Allow SSH (port 22)
   - Allow HTTP (port 80)
   - Allow HTTPS (port 443)
   - Allow custom TCP 3000

### Step 2: Connect to Server
```bash
ssh -i your-key.pem ubuntu@your-ec2-ip
```

### Step 3: Install Dependencies
```bash
sudo apt update
sudo apt install nodejs npm mysql-server -y

# Verify
node --version
npm --version
mysql --version
```

### Step 4: Clone Repository
```bash
git clone https://github.com/haadymebron/ecommerce-site.git
cd ecommerce-site/server
npm install
```

### Step 5: Configure Environment
```bash
nano .env
```
Add:
```
PORT=3000
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your-password
DB_NAME=ecommerce_db
JWT_SECRET=your-secret
NODE_ENV=production
```

### Step 6: Start MySQL
```bash
sudo mysql
```
```sql
CREATE DATABASE ecommerce_db;
CREATE USER 'root'@'localhost' IDENTIFIED BY 'your-password';
GRANT ALL PRIVILEGES ON ecommerce_db.* TO 'root'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### Step 7: Run Backend
```bash
npm start
```

### Step 8: Setup PM2 (Keep Running)
```bash
sudo npm install -g pm2
pm2 start server.js --name "ecommerce-api"
pm2 startup
pm2 save
```

### Step 9: Setup Nginx Reverse Proxy
```bash
sudo apt install nginx -y
sudo nano /etc/nginx/sites-available/default
```

Replace with:
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

```bash
sudo systemctl restart nginx
```

### Step 10: Get SSL Certificate
```bash
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d your-domain.com
```

## **Environment Variables Explanation**

```env
# Server Config
PORT=3000                    # Port to run on
NODE_ENV=production          # Set to production for deployment

# Database
DB_HOST=localhost            # Database server address
DB_USER=root                 # Database username
DB_PASSWORD=your-password    # Database password
DB_NAME=ecommerce_db         # Database name

# Security
JWT_SECRET=your-very-secret-key-min-32-chars-long  # JWT encryption key
```

## **Recommended Deployment Stack**

For your college marketplace:

1. **Frontend:** Vercel (free, fast CDN)
2. **Backend:** Railway (free tier, includes MySQL)
3. **Domain:** Namecheap ($1-3/year)

## **Database Connection Setup**

### For Railway MySQL:
```env
DB_HOST=mysql.railway.internal
DB_USER=root
DB_PASSWORD=<auto-generated-check-variables>
DB_NAME=railway
```

### For External MySQL:
```env
DB_HOST=db.example.com
DB_USER=admin
DB_PASSWORD=your-password
DB_NAME=ecommerce_db
```

### Verify Connection:
```bash
# From terminal
mysql -h <DB_HOST> -u <DB_USER> -p<DB_PASSWORD> -D <DB_NAME>

# Should show:
mysql> 
```

## **Troubleshooting**

### Backend Not Starting
```bash
# Check logs
heroku logs --tail  # Heroku
railway run npm start  # Railway
pm2 logs  # PM2
```

### Database Connection Error
```bash
# Verify credentials
mysql -u root -p
SHOW DATABASES;
USE ecommerce_db;
SHOW TABLES;
```

### CORS Errors
Backend already has CORS enabled. If errors:
```javascript
// In server.js
app.use(cors({
  origin: 'https://your-frontend.vercel.app',
  credentials: true
}));
```

### 502 Bad Gateway
- Check backend is running: `PORT=3000 npm start`
- Check database connection
- Review error logs
